Vial 改键教程

这份教程面向日常使用，按顺序完成后你就能掌握 Vial 的核心改键流程：普通改键、宏、Tap Dance、Combo、Key Override、Alt Repeat。

准备工作

  1. 准备一根可传输数据的 Type-C 线（仅充电线可能无法识别）。
  2. 建议先导出当前配置做备份，避免误操作后找不回原设置。
  3. 确认键盘固件已启用 Vial 功能并且可以正常连接。

提示：如果你在 Vial 中看不到某些标签页（例如 Combo/Tap Dance），通常是固件未启用对应功能，而不是软件故障。

识别键盘

使用 Type-C 数据线连接键盘，设备管理器里面能看到对应键盘设备。

示例：连接后在设备管理器中看到 ZOROKB Keyboard（名称可能略有不同），说明系统已识别硬件。

安装 Vial

下载 Vial 或者使用 Vial 网页版 进行连接

示例：在公司电脑无安装权限时，直接使用 Vial 网页版 完成连接与改键。

打开 Vial

  1. 打开 Vial 后，左上角能看到你的键盘名称，说明连接成功。
  2. 先确认可以在 Keymap 页点击按键并选中目标按键位。
  3. 点击顶部 File 菜单，选择 Load saved layout... 导入配置数据。
  4. 选择其他用户导出的布局文件后确认导入，键盘会加载对应按键布局。
  5. 如果对方给的是 JSON 文件，可使用 Sideload VIA JSON... 导入。
  6. 导入完成后测试常用键位、切层键和连接模式键，确认与你的键盘功能一致。

示例：页面左上角显示 zorokb Tumblor40 后，通过 File -> Load saved layout... 导入好友布局，再做少量个性化调整。

修改按键

  1. 进入 Keymap 页面，点击你要修改的键位。
  2. 在下方功能区选择目标键值（如字母、功能键、媒体键等）。
  3. 修改后立即生效，按下实体键即可验证是否正确。

示例：把 Caps Lock 改成 Esc，保存后按原 Caps 键会输出 Esc。

修改布局

  1. 进入 Layout 页面，根据键盘实际配列开启/关闭对应选项。
  2. 完成后回到 Keymap 检查键位图是否与实物布局一致。
  3. 如果布局变化较大，建议重新检查每一层的关键按键位置。

示例：你实际使用分裂空格，就在 Layout 中启用 split space，再回到 Keymap 给左右空格分别分配不同功能。

Layers 页面按键说明

n 表示层号（0~11），例如 MO(1) 就是操作第 1 层。

  • Fn1(Fn3) / Fn2(Fn3)

    兼容旧式 Fn 动作入口，具体行为由固件里对应函数动作决定。

    示例：固件把 Fn3 定义成切层时，按该键会执行对应 Fn 层动作。

  • MO(n)

    按住临时进入第 n 层，松开立刻返回。

    示例：MO(1) 常用作 Fn 键。

  • DF(n) / PDF(n)

    DF(n) 设置默认层；PDF(n) 为可持久化默认层（重启后仍保留）。

    示例：把 PDF(1) 设到某键，按一次后默认层长期切到 Layer 1。

  • TG(n)

    切换第 n 层开/关，按一次开，再按一次关。

    示例：TG(2) 适合临时开启数字层。

  • TT(n)

    短按多次可切换层，长按则临时进层（兼具 Tap Toggle 与 MO 逻辑）。

    示例：长按 TT(1) 当 Fn；快速连按达到阈值可锁定 Layer 1。

  • OSL(n)

    单次切层：按下后只对“下一次按键”生效一次。

    示例：OSL(1) 后再按一个符号键，执行完自动回主层。

  • TO(n)

    直接跳转到第 n 层并退出其它临时层状态。

    示例：设置 TO(0) 作为“一键回主层”。

  • LT n (kc)

    双功能键：短按输出 kc，长按进入第 n 层。

    示例：LT 1 (Space)，点按是空格，长按进 Layer 1。

推荐新手先用 MO(n) + TO(0) 跑通流程，再升级到 LT n (kc) 做键位压缩。

修饰键页面按键说明

常见缩写：C=Ctrl、S=Shift、A=Alt、G=GUI(Win/Command)。

  • OSM ...

    One Shot Mod，一次性修饰键。按一次后，只作用于下一次按键。

    示例：OSM LSft 后再按 a，只这一次输出大写 A。

  • LSft(kc) / LCtl(kc) / LAlt(kc) / LGui(kc)

    发送“修饰键 + kc”的组合输出。

    示例：LCtl(kc) + C 可做成一键 Ctrl+C。

  • LCS(kc) / LCA(kc) / LCG(kc) / RCG(kc) / LSA(kc) / LSG(kc) / LCAG(kc)

    多修饰组合版本，按下即发送“多修饰 + kc”。

    示例：LCS(kc) + Z 可快速触发 Ctrl+Shift+Z。

  • Meh(kc) / Hyper(kc)

    Meh 通常为 Ctrl+Shift+Alt；Hyper 通常为 Ctrl+Shift+Alt+GUI。

    示例：给自动化软件设置 Hyper+J 作为全局快捷键，和常规快捷键冲突更少。

  • LSft_T(kc) / LCtl_T(kc) / ... / RGui_T(kc)

    Mod-Tap：短按输出 kc，长按当修饰键使用。

    示例：LCtl_T(Esc)，点按是 Esc，长按是 Ctrl。

  • LCS_T(kc) / LCA_T(kc) / ... / LCAG_T(kc) / RCAG_T(kc) / Meh_T(kc) / ALL_T(kc)

    组合修饰的 Mod-Tap 版本：短按是 kc，长按是对应组合修饰。

    示例：Meh_T(Tab)，点按 Tab，长按当 Meh 修饰键。

实战建议：日常输入优先从 OSM 和单修饰 _T(kc) 开始，确认不误触后再上组合修饰 LCAG_T/Meh_T/ALL_T。

设置宏

  1. 进入 Macros 页面，选择宏槽 M0*。
  2. 右下角Add action按钮顺序添加动作：Text、Down、Up、Delay (ms)、Tap。
  3. 把 Text 内容填写为 zorokb。
  4. 把 Down 设为 A，把 Up 也设为 A。
  5. 把 Delay (ms) 设为 200。
  6. 把 Tap 设为 B。
  7. 回到 Keymap，将某个按键绑定为 M0 后测试输出。
M0 动作顺序
1) Text "zorokb"
2) Down A
3) Up A
4) Delay 200ms
5) Tap B

输出案例：zorokbAB，其中 A 与 B 之间有约 200ms 延迟。

建议：宏里尽量避免过长延时；涉及密码、令牌等敏感信息不要写进宏。

设置 Tap Dance

  1. 进入 Tap Dance 页面，选择槽位 0*（即 TD(0)）。
  2. 按图设置 4 个动作：On tap = A、On hold = B、On double tap = C、On tap + hold = D。
  3. 把 Tapping term (ms) 设为 200ms。
  4. 点击 Save 保存。
  5. 回到 Keymap，把目标按键设置为 TD(0) 并测试。
TD(0) 参数
On tap        -> A
On hold       -> B
On double tap -> C
On tap + hold -> D
Tapping term  -> 200ms

输出案例：单击触发 A，按住触发 B，双击触发 C，点按后继续按住触发 D。识别窗口为约 200ms。

设置 Combo

  1. 进入 Combos 页面，选择一个组合槽位。
  2. 设置 Key 1 = Q、Key 2 = W、Key 3 = E，Key 4 保持空白。
  3. 设置 Output key = R。
  4. 在键盘上几乎同时按下 Key 1、Key 2、Key 3 进行测试。
Combo 参数
Key 1      -> Q
Key 2      -> W
Key 3      -> E
Key 4      -> (空)
Output key -> R

输出案例（按图中配置）：同时按下 Q + W + E 触发组合输出 R；若不是同时按下（间隔时间大于 50ms），则通常按各自原键输出。

建议：先用两键 Combo 验证逻辑，再逐步增加更复杂组合，能更快排查误触问题。

设置 Key Overrides

Key Override 的核心作用：当“触发键 + 指定修饰键”满足条件时，把原本要输出的键替换成另一个键。

  • Enable

    当前这条规则的总开关。关闭后该规则完全不生效。

  • Enable on layers

    这条规则在哪些层生效。勾选 0~15 中的层号；只在被勾选的层触发。Enable all/Disable all 用于一键全选/全不选。

  • Trigger

    触发键本体（图中是 A）。只有按下这个键才会评估规则。

  • Trigger mods

    必要修饰键。这里勾选的修饰键必须被按住，规则才触发（图中勾选了 LShift）。

  • Negative mods

    禁止修饰键。这里勾选的修饰键一旦被按住，就不允许触发这条规则。

  • Suppressed mods

    触发后要“抑制”的修饰键。用于避免修饰键继续影响最终输出。

  • Replacement

    替换输出键（图中是 B）。满足条件时，输出它而不是 Trigger 键原本的输出。

  • Options

    规则激活/失活时机的细节开关。用于控制按下、释放、连按时的边界行为。

图中案例（建议先照着验证）
Enable        -> 开
Layer         -> 勾选目标层（示例常用 Layer 0）
Trigger       -> A
Trigger mods  -> LShift
Negative mods -> (空)
Suppressed mods -> (空)
Replacement   -> B
Options       -> 可见前3项已勾选（其余保持默认）

输出案例（按图中思路）：按住 LShift 再按 A 时，输出会被改写为 B。如果没按住 LShift，则按键按原逻辑输出。

Options 每一项含义：

Activate when the trigger key is pressed down按下触发键时就尝试激活。

Activate when a necessary modifier is pressed down按下必要修饰键时可立即进入可触发状态。

Activate when a negative modifier is released负向修饰键释放后可重新满足触发条件。

Activate on one modifier只要命中一个修饰键条件即可触发（不常用，建议谨慎开启）。

Don't deactivate when another key is pressed down按其他键时不立刻失活，适合连续组合输入。

Don't register the trigger key again after the override is deactivated规则结束后不再补发 Trigger 键原始按键，避免多余字符。

设置 Alt Repeat Key

Alt Repeat Key 用于“基于上一次按键输出替代键”。常见用法是：最近按了某个键后，再按 Alt Repeat 键输出你定义的替代键。

  • Enable

    当前这条 Alt 规则开关。关闭后这条映射不参与计算。

  • Last key

    最近一次按下的目标键（图中是 A）。

  • Alt key

    替代输出键（图中是 B）。当 Last key 条件命中时，Alt Repeat 键会输出它。

  • Allowed mods

    允许继承的修饰键。图中勾选了 LShift，表示在该修饰键状态下也允许这条映射生效。

  • Options

    行为细节开关，决定默认回退、双向映射、左右手修饰键匹配方式。

图中案例（建议先照着验证）
Enable       -> 开
Last key     -> A
Alt key      -> B
Allowed mods -> LShift
Options      -> 默认未勾选（按图）

输出案例（按图中思路）：最近一次主键是 A 时，按下 Alt Repeat 键将输出 B；如果当前修饰键条件不在允许范围内，则按默认逻辑处理。

Options 每一项含义：

Default to this alt key当没有匹配到 Last key 映射时，默认也输出当前 Alt key。

Bidirectional启用双向映射（例如 A→B 同时允许 B→A 的反向关系，具体表现以固件实现为准）。

Ignore mod handedness忽略修饰键左右手差异（例如把 LShift / RShift 视为同类匹配）。

设置 Settings

本节按你截图中的 QMK Settings 页面逐行解释，每个参数后都带一个可直接照抄的案例。最后统一点 Save 保存；要撤销当前未保存改动，点 Reset。

1. 按键交换与基础行为

  • Swap Caps Lock and Left Control

    交换 Caps Lock 和左 Ctrl 的功能。

    案例：开启后原本 Caps 位置变成左 Ctrl，更适合终端快捷键。

  • Treat Caps Lock as Control

    把 Caps Lock 当成 Ctrl，但不和左 Ctrl 互换。

    案例：你几乎不用大写锁定，开启后可用 Caps 位做 Ctrl + C。

  • Swap Left Alt and GUI

    交换左 Alt 与左 GUI/Win/Command。

    案例：Mac 用户想要左下角更接近 Command 手感时可开启。

  • Swap Right Alt and GUI

    交换右 Alt 与右 GUI。

    案例：你在右手区常用系统快捷键，开启后可把右 GUI 放到更顺手位置。

  • Disable the GUI keys

    禁用所有 GUI 键。

    案例：游戏时避免误触 Win 键弹出系统菜单，可临时开启。

  • Swap ` and Escape

    交换 ` 与 Esc。

    案例：Vim 用户把更易按的位置给 Esc，减少误按。

  • Swap \\ and Backspace

    交换 \\ 与 Backspace。

    案例：40/60 配列里想让退格更近，开启后原反斜杠位可直接退格。

  • Enable N-key rollover

    开启全键无冲（NKRO）。

    案例：需要同时按很多键（游戏或和弦输入）时，开启可减少按键冲突。

  • Swap Left Control and GUI

    交换左 Ctrl 与左 GUI。

    案例：Windows 键位迁移到 Mac 习惯时常用。

  • Swap Right Control and GUI

    交换右 Ctrl 与右 GUI。

    案例：右手区常用快捷键时，把右 GUI 调整到右 Ctrl 原位置。

2. Esc 发送策略

  • Always send Escape if Alt is pressed

    按 Esc 时若 Alt 处于按下状态，也强制输出 Esc。

    案例：你按了 Alt 后再按 Esc，依然稳定输出 Esc，不被 Alt 组合干扰。

  • Always send Escape if Control is pressed

    按 Esc 时若 Ctrl 按下，也强制输出 Esc。

    案例：终端中手指还压着 Ctrl 时，Esc 仍可正常退出模式。

  • Always send Escape if GUI is pressed

    按 Esc 时若 GUI 按下，也强制输出 Esc。

    案例：按着 Win/Command 误触 Esc，依旧只发 Esc。

  • Always send Escape if Shift is pressed

    按 Esc 时若 Shift 按下，也强制输出 Esc。

    案例：双手切输入状态时即使压着 Shift，Esc 也不会失效。

3. Tap-Hold 与敲击窗口

  • Tapping Term

    Tap/Hold 判定窗口，图中为 200ms。

    案例：双功能键在 200ms 内松开判定为 Tap，超过判定为 Hold。

  • Permissive Hold

    在特定连按场景更倾向判定为 Hold。

    案例：你先按住双功能键再快速按别键，开启后更容易触发层/修饰键逻辑。

  • Hold On Other Key Press

    只要期间按了其他键，优先判定为 Hold。

    案例：按住双功能键同时敲字母时，稳定触发 Hold，不误出 Tap 字符。

  • Retro Tapping

    先按住后直接松开也可能回补为 Tap。

    案例：你轻按略慢，开启后可减少“本想单击却变按住”的误判。

  • Quick Tap Term

    快速连击判定时间，图中为 200ms。

    案例：同一键在 200ms 内重复敲击，按快速敲击策略处理。

  • Tap Code Delay

    发送 tapcode 间隔，图中为 10ms。

    案例：宏里连续发送按键时加 10ms，可提升部分系统下的识别稳定性。

  • Tap Hold Caps Delay

    Tap-Hold 与 Caps 相关延迟，图中为 80ms。

    案例：Caps 参与双功能配置时，设置 80ms 可降低误触。

  • Tapping Toggle

    连续敲击达到次数后保持激活，图中为 5。

    案例：连续点同一功能键 5 次后进入保持状态，再点一次退出。

  • Chordal Hold

    和弦按法下的 Hold 优化开关。

    案例：双手和弦输入较多时开启，可减少 Tap/Hold 冲突。

  • Flow Tap

    连贯输入的 Tap 流控参数，图中为 0。

    案例：设为 0 表示关闭该流控，先用默认行为调试最稳妥。

4. Auto Shift

  • Enable

    Auto Shift 总开关。

    案例：开启后，短按输出小写，按住超过阈值可直接输出大写或符号。

  • Enable for modifiers

    是否对修饰键相关情形启用 Auto Shift。

    案例：你希望带修饰键组合时也保持 Auto Shift 行为，可开启。

  • Timeout

    触发 Auto Shift 的按住阈值，图中为 175ms。

    案例：按住字母超过 175ms 输出大写。

  • Do not Auto Shift special keys

    特殊键不参与 Auto Shift。

    案例：功能键或控制键不做长按变体，避免异常输出。

  • Do not Auto Shift numeric keys

    数字键不参与 Auto Shift。

    案例：你只希望字母受 Auto Shift 影响，数字始终原样输出。

  • Do not Auto Shift alpha characters

    字母不参与 Auto Shift。

    案例：只想对数字/符号做长按变化时开启此项。

  • Enable keyrepeat

    允许长按重复发送。

    案例：按住方向键持续移动光标时可用。

  • Disable keyrepeat when timeout is exceeded

    超过阈值后禁止重复发送。

    案例：防止长按字母在触发 Auto Shift 后继续刷屏。

5. Combo 全局时间

  • Time out period for combos

    Combo 的同时按下窗口，图中为 50ms。

    案例：必须在约 50ms 内一起按下组合键才触发 Combo，否则按普通按键输出。

6. One Shot

  • Tapping this number of times holds the key until tapped once again

    达到该连击次数后进入“锁定保持”，图中为 5。

    案例：One Shot Shift 连点 5 次后保持 Shift 状态，再点一次解除。

  • Time (in ms) before the one shot key is released

    One Shot 自动释放时间，图中为 5000ms。

    案例：触发 One Shot 后若 5000ms 内无后续按键则自动失效。

7. Mouse Keys（鼠标键）

  • Delay between pressing a movement key and cursor movement

    按下移动键到光标开始动的延迟，图中为 10ms。

    案例：设为 10ms 后光标几乎立即开始移动。

  • Time between cursor movements in milliseconds

    两次光标位移间隔，图中为 20ms。

    案例：值越小，移动更新越频繁，手感更连贯。

  • Step size

    每次移动步长，图中为 8。

    案例：从 8 提到 12，单步位移更大，适合高分辨率屏幕。

  • Maximum cursor speed at which acceleration stops

    加速停止前的最大光标速度，图中为 10。

    案例：设为 10 可避免长按后速度无限变快难以控制。

  • Time until maximum cursor speed is reached

    达到最大光标速度所需时间，图中为 30ms。

    案例：设短一些可更快冲到高速，适合大范围移动。

  • Delay between pressing a wheel key and wheel movement

    滚轮键按下到滚动开始延迟，图中为 10ms。

    案例：设为 10ms 后滚动响应更快。

  • Time between wheel movements

常见问题

  • 看不到某个功能页：检查固件是否开启该功能并重新刷写。
  • 改键后没生效：确认当前层（Layer）是否正确，目标键是否在该层被覆盖。
  • Tap Dance 误触：适当提高 Tapping term 并减少重叠功能。
  • Combo 易冲突：缩短 Combo 时间窗口，或更换触发按键组合。
  • 建议在每次大改前先备份，出问题可快速回滚。
