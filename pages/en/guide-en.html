Vial Remapping Guide

This guide focuses on practical daily usage. Follow it in order and you will understand the core Vial workflow: basic remapping, macros, Tap Dance, Combo, Key Override, and Alt Repeat.

Preparation

  1. Prepare a USB Type-C cable that supports data transfer. Charge-only cables may not work.
  2. Export your current layout before making changes so you can roll back if needed.
  3. Make sure your firmware has Vial enabled and the keyboard can connect correctly.

Tip: if some tabs such as Combo or Tap Dance do not appear in Vial, the firmware likely does not enable them.

Identify the Board

Connect the keyboard with a Type-C data cable and check that the operating system detects the device.

Example: if Device Manager shows something like ZOROKB Keyboard, the hardware connection is working.

Install Vial

Download Vial or connect through the Vial web app

Example: if you do not have install permissions on a work machine, use the web app to connect and edit keys.

Open Vial

  1. After opening Vial, look at the top-left corner. If your keyboard name appears, the connection is successful.
  2. Open the Keymap page and confirm you can select a key on the layout.
  3. Use File → Load saved layout... to import a layout file when needed.
  4. If another user shared a JSON layout, you can also use Sideload VIA JSON....
  5. After importing, test your important keys, layer keys, and connection mode keys.

Example: once the top-left area shows zorokb Tumblor40, you can load a shared layout and then make your own adjustments.

Change Keys

  1. Open the Keymap page and click the key you want to change.
  2. Select the target keycode from the area below, such as letters, function keys, or media keys.
  3. The change takes effect immediately. Press the physical key to confirm the result.

Example: remap Caps Lock to Esc. After saving, pressing Caps outputs Esc.

Change Layout

  1. Open the Layout page and enable the options that match your actual physical layout.
  2. Return to Keymap and confirm the visual layout matches the board.
  3. If the layout changes significantly, re-check important keys on every layer.

Example: if your board has split space, enable it on the layout page and then assign separate functions to both space keys.

Layer Page Key Types

n means layer number, usually from 0 to 11. For example, MO(1) targets layer 1.

  • MO(n)

    Momentarily enter layer n while the key is held.

    Example: MO(1) is commonly used as an Fn key.

  • DF(n) / PDF(n)

    Set the default layer. PDF persists after reboot.

    Example: assign PDF(1) to switch the board to layer 1 permanently.

  • TG(n)

    Toggle layer n on and off.

    Example: TG(2) is useful for a temporary number layer.

  • TT(n)

    Tap to toggle after a threshold, hold to use momentarily.

    Example: long-hold as Fn, repeated taps to lock the layer.

  • OSL(n)

    One-shot layer. The next key uses that layer once.

    Example: press OSL(1), then a symbol key, and the keyboard returns automatically.

  • TO(n)

    Jump directly to layer n and leave temporary layer states.

    Example: use TO(0) as a one-key “return to base layer”.

  • LT n (kc)

    Tap for kc, hold for layer n.

    Example: LT 1 (Space) sends Space on tap and enters layer 1 on hold.

A good beginner path is MO(n) plus TO(0), then later moving to LT n (kc).

Modifier Page Key Types

Common abbreviations: C = Ctrl, S = Shift, A = Alt, G = GUI.

  • OSM ...

    One Shot Mod. It only affects the next key press.

    Example: OSM LSft then a outputs a single uppercase A.

  • LSft(kc) / LCtl(kc) / LAlt(kc) / LGui(kc)

    Send a modifier plus a normal key.

    Example: use a shortcut key for Ctrl+C.

  • Meh(kc) / Hyper(kc)

    Meh usually means Ctrl+Shift+Alt, and Hyper adds GUI as well.

    Example: Hyper shortcuts are useful for global automation without conflicts.

  • LSft_T(kc) / LCtl_T(kc)

    Mod-Tap. Tap sends kc, hold acts as the modifier.

    Example: LCtl_T(Esc) taps Esc, holds Ctrl.

  • Meh_T(kc) / ALL_T(kc)

    Tap for a normal key, hold for a combined modifier.

    Example: Meh_T(Tab) taps Tab and holds Meh.

A practical starting point is OSM and simple Mod-Tap keys before using heavy combined modifiers.

Set Up Macros

  1. Open the Macros page and choose a slot such as M0*.
  2. Add actions in order: Text, Down, Up, Delay (ms), Tap.
  3. Fill the text field with zorokb.
  4. Set Down to A and Up to A.
  5. Set the delay to 200.
  6. Set Tap to B.
  7. Return to Keymap, bind a key to M0, and test it.
M0 action order
1) Text "zorokb"
2) Down A
3) Up A
4) Delay 200ms
5) Tap B

Example output: zorokbAB, with about 200ms between A and B.

Avoid long macro delays where possible, and do not store passwords or tokens in macros.

Set Up Tap Dance

  1. Open the Tap Dance page and choose slot 0*, which corresponds to TD(0).
  2. Configure four actions: On tap = A, On hold = B, On double tap = C, On tap + hold = D.
  3. Set Tapping term (ms) to 200ms.
  4. Click Save.
  5. Return to Keymap, assign TD(0), and test it.
TD(0) settings
On tap        -> A
On hold       -> B
On double tap -> C
On tap + hold -> D
Tapping term  -> 200ms

Example behavior: single tap sends A, hold sends B, double tap sends C, tap-hold sends D.

Set Up Combo

  1. Open the Combos page and choose a combo slot.
  2. Set Key 1 = Q, Key 2 = W, Key 3 = E, leaving Key 4 empty.
  3. Set Output key = R.
  4. Press the combo keys almost simultaneously to test it.
Combo settings
Key 1      -> Q
Key 2      -> W
Key 3      -> E
Key 4      -> (empty)
Output key -> R

Example: press Q + W + E together to trigger R. If the time gap is larger than 50ms, the keys usually register individually.

Start with simple two-key combos before building more complex ones.

Set Up Key Override

Key Override replaces one output with another when the trigger key and required modifiers match the rule.

  • Enable

    Master switch for the current rule.

  • Enable on layers

    Choose which layers the rule applies to.

  • Trigger

    The key that starts the rule, such as A.

  • Trigger mods

    Required modifiers, such as LShift.

  • Negative mods

    Modifiers that block the rule when held.

  • Suppressed mods

    Modifiers to suppress after the override triggers.

  • Replacement

    The output key that replaces the trigger, such as B.

  • Options

    Behavior details that control when the rule activates or deactivates.

Example setup
Enable          -> On
Layer           -> choose the target layer, often Layer 0
Trigger         -> A
Trigger mods    -> LShift
Negative mods   -> none
Suppressed mods -> none
Replacement     -> B

Example: hold LShift and press A, and the output becomes B. Without the required modifier, A behaves normally.

Set Up Alt Repeat Key

Alt Repeat outputs an alternate key based on the previous key that was pressed.

  • Enable

    Enables or disables this Alt Repeat rule.

  • Last key

    The preceding key that must match, such as A.

  • Alt key

    The alternative key that gets output, such as B.

  • Allowed mods

    Modifiers that are allowed to remain active when this mapping runs.

  • Options

    Extra behavior such as default fallback, bidirectional mapping, or modifier handedness matching.

Example setup
Enable       -> On
Last key     -> A
Alt key      -> B
Allowed mods -> LShift

Example: if the most recent key was A, pressing the Alt Repeat key outputs B.

3. Tap-Hold windows

  • Tapping Term

    Tap-vs-hold window, shown as 200ms in the screenshot.

  • Permissive Hold

    Bias certain multi-key flows toward hold behavior.

  • Hold On Other Key Press

    Pressing another key during the window forces hold behavior.

  • Retro Tapping

    Allows a slow single tap to fall back to tap when released.

  • Quick Tap Term

    Window for rapid repeated taps, shown as 200ms.

  • Tap Code Delay

    Delay between generated tapcodes, shown as 10ms.

  • Tap Hold Caps Delay

    Caps-related tap-hold delay, shown as 80ms.

  • Tapping Toggle

    Number of taps needed to lock a tap-toggle key, shown as 5.

  • Chordal Hold

    Hold optimization for chord-like usage.

  • Flow Tap

    Flow-tap parameter, shown as 0 in the screenshot.

4. Auto Shift

  • Enable for modifiers

  • Timeout

  • Do not Auto Shift alpha characters

  • Enable keyrepeat

FAQ

  • If a function page is missing, check whether the firmware enables that feature and reflash if needed.
  • If a remap does not work, verify the current layer and whether another layer overrides the target key.
  • If Tap Dance misfires, increase Tapping term and reduce overlapping behaviors.
  • If combos conflict too often, shorten the combo window or change the triggering keys.
  • Back up your layout before major edits so you can recover quickly.
